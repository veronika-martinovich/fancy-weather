History web API

What is it?


The History API is the way browsers let you:

  • interact with the browser history
  • trigger the browser navigation methods
  • change the address bar content

Fragment ID:

    window.location.hash

History API vs. Fragmented hash URL

What is common

  • both do not reload a page while navigating the app to speed up the application
  • both create browser history
Pros to use History API
  • it looks cleaner (compare two urls): www.example.com\foo.html#bar www.example.com\bar
  • SEO is supported
  • you can see the full URL on your server (unlike with # url)
  • you can use hash tag for what it was meant for (as basic web anchors to scroll down the page)

Some cons to use History API


  • to bookmark or copy link additional server-side actions will be required
  • some old browsers will need polyfill

History API

Details

Access



						window.history || history

History object is unique for each browser tab

Navigate the history


Go back to the previous page:

history.back()

Forward to the next page:

history.forward()

Navigate back or forward multiple levels deep:


						history.go(-1) //equivalent to history.back()
						history.go(-2) //equivalent to calling history.back() twice
						history.go(1) //equivalent to history.forward()
						history.go(3) //equivalent to calling history.forward() 3 times

Check how many entries there are in the history:

history.length

Add an entry to the history

history.pushState()

Accepts 3 parameters

  • 1 state object which can contain anything

    Limitations:

    - size limit 640Kb

    - the object needs to be serializable

  • 2 currently unused (pass an empty string)

  • 3 URL associated to the new state

Example:


						// URL: https://example.com

						const state = { name: 'Flavio' }
						history.pushState(state, '', '/user') // https://example.com/users

Modify history entries

history.replaceState()

Example:


						history.pushState({}, '', '/posts')

						const state = { post: 'first' }
						history.pushState(state, '', '/post/first')

						const state = { post: 'second' }
						history.replaceState(state, '', '/post/second')

						history.back() //  /posts

Access the current history entry state


history.state



						const state = { post: 'first' }
						history.pushState(state, '', '/post/first')

						history.state // { post: 'first' }

Neither history.pushState() nor history.replaceState() won’t change the content of the page, and will not cause any browser action.
